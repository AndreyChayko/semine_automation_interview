Expense Tracking Demo — Test Brief

Overview

Semine is a single-page Angular application that simulates a corporate expense management tool. It is purpose-built as an interview exercise to evaluate test automation skills across authentication, stateful UI flows, runtime configuration, and data visualisation.

All back-end interactions are emulated by a mock API service with configurable RxJS delays, so the app behaves like a real network-connected system without any server dependency.

Application Pages

Login /login

Email + password form with validation, error display, show/hide password, and one-click credential fill buttons for each demo account.

Profile /profile

Displays user info and allows editing the profile. Changes are persisted to sessionStorage and reflected in the navbar immediately.

Expenses /expenses

Timeline view of all expenses grouped by date. Each login seeds 1–3 paid expenses. New expenses are created via a modal form and have pending status. Pending expenses can be deleted; paid ones are immutable. A monthly budget bar shows current spend vs. the configured cap.

Expense History /expense-history

Bar charts showing spending by category. The current month reflects live data from the Expenses page. The past 6 months are generated pseudo-randomly per user on first login and cached in sessionStorage. Layout switches between list and grid via the DEV panel.

Key User Flows

Authentication

  1. Navigate to /login (unauthenticated users are redirected here by guard).
  2. Enter credentials or use the Use this account buttons.
  3. The form validates inline; server errors appear below the submit button.
  4. On success, the user is redirected to /profile and the session timer starts.
  5. Logging out clears all in-memory state and returns to /login.

Session Management

  1. A countdown timer in the navbar shows time remaining (HH:mm:ss).
  2. Colour transitions: green → amber (40 % remaining) → red (10 % remaining).
  3. A warning toast fires timeBeforeExpiredTrigger ms before expiry.
  4. On expiry the session ends and the user is automatically logged out.
  5. The session state survives page refresh (stored in sessionStorage) but not a new tab.

Adding an Expense

  1. Click Add Expense on the Expenses page (disabled when quota is exceeded).
  2. Fill in title, amount, currency, category, and date in the modal.
  3. The new expense appears in the timeline with PENDING status.
  4. The monthly budget bar updates in real time.
  5. Hover over a pending expense to reveal the delete button.

Expense History

  1. Navigate to /expense-history via the sidebar (requires login).
  2. The current month chart is always live — it reflects every expense added or deleted on the Expenses page in real time.
  3. The past 6 months are generated pseudo-randomly per user on first login and cached in sessionStorage; they differ between Alice and Bob.
  4. Each bar chart groups spending by category (Travel, Meals, Software, Office, Training, Other).
  5. The layout can be toggled between list (single column) and grid (two columns on tablet+, always list on mobile) via DEV panel → history → layout.
  6. If history.enabled is turned off in the DEV panel, the page shows a locked placeholder instead of charts.

DEV Panel — Runtime Configuration

Click the DEV tab on the right edge of the screen. Settings are stored per-user in sessionStorage and take effect immediately after Save & Close. Reset restores defaults without needing to save. The panel is accessible before login.

auth

  • sessionTime — total session length in ms (default 5 min).
  • timeBeforeExpiredTrigger — ms before expiry to show warning toast (0 = disabled).
  • triggerMessage — custom text for the warning toast.

expenses

  • enabled — toggles the Expenses page feature. Off shows a locked placeholder.
  • maxMonthlyAmount — spending cap; disables Add Expense and shows a quota banner when exceeded.

history

  • enabled — toggles the Expense History page. Off shows a locked placeholder.
  • layout — list (single column) or grid (2 columns on tablet+; always list on mobile).
